TOHSENO / PRIVACY

Privacy notice

This website

tohseno.com serves static pages and installer scripts. It has no accounts, no forms, no cookies, no analytics, and no third-party browser resources. The server keeps standard operational request logs (route, status, timing) with no tracking identifiers.

The installer and local factory

The installer sends no telemetry and accepts no credentials. It downloads only pinned release artifacts, verifies SHA-256 checksums, and installs under ~/.tohseno. The deprecated oneshot URL creates nothing and points to this canonical path.

TOHSENO Studio runs on the Mac and binds to 127.0.0.1. Intentions, Markdown, reference images, shots, builds, and Simulator video stay local; Studio does not upload them to tohseno.com. The browser copies creation inputs into an isolated local staging directory, then removes that staging data after the factory finishes or fails safely.

Each new shot keeps its normalized intention and reference provenance inside its own gitignored .tohseno/provenance directory so the owner can move the complete shot between their machines. That private input is not committed to the shot’s Git history or written to content-bearing factory logs. Anyone given the whole unfiltered shot directory may also receive that provenance, so owners should inspect it before sharing an archive.

Apps built on these rails

A continuity app is private by default: identity is a locally generated seed phrase, and writing lives on the device as plain files. Nothing leaves the device unless the app's manifest declares it. The baseline shot API receives health requests and stores operational metadata only; TOHSENO operates no centralized backend for generated apps and never receives their users' content. An explicitly requested Quick Tunnel makes that development API publicly reachable at a random URL until stopped, but it is never a production endpoint. Every app builds and runs without TOHSENO credentials.
