TOHSENO / PRIVACY

Privacy notice

No accounts, analytics, or content backend. The factory runs locally; generated app content stays on the person’s device.

This website

tohseno.com serves static pages and installer scripts. It has no accounts, no forms, no cookies, no analytics, and no third-party browser resources. The server keeps standard operational request logs (route, status, timing) with no tracking identifiers.

The installer and local factory

The installer sends no telemetry and accepts no credentials. It downloads only pinned release artifacts, verifies the archive and its complete internal inventory, and installs under ~/.tohseno. Its wrapper rechecks managed CLI and runtime bytes before execution. The deprecated oneshot URL creates nothing and points to this canonical path.

TOHSENO Studio runs on the Mac and binds to 127.0.0.1. It opens through an owner-only temporary launcher and requires a private local browser session for every shot read or mutation. Intentions, Markdown, reference images, shots, builds, and Simulator video are not uploaded to tohseno.com. The browser copies creation inputs into an isolated local staging directory, then removes that staging data after the factory finishes or fails safely.

Each new shot keeps its normalized intention and reference provenance inside its own gitignored .tohseno/provenance directory so the owner can move the complete shot between their machines. That private input is not committed to the shot’s Git history or written to content-bearing factory logs. The verifier checks the worktree after every coding-agent exit and isolates an unsafe result. Anyone given the whole unfiltered shot directory may also receive private provenance, so owners should inspect it before sharing an archive.

TOHSENO itself does not send creation input to a TOHSENO service. A coding agent selected by the owner can read that input to build the app; its provider is a separate trust boundary with its own account, privacy, and retention terms.

Apps built on these rails

A continuity app is private by default: identity is a locally generated seed phrase, and writing lives on the device as plain files. Nothing leaves the device unless the app's manifest declares it. The baseline shot API receives health requests and stores operational metadata only; TOHSENO operates no centralized backend for generated apps and never receives their users' content. An explicitly requested Quick Tunnel makes that development API publicly reachable at a random URL until stopped, but it is never a production endpoint. Every app builds and runs without TOHSENO credentials.
