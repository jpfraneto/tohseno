From one prompt to an iOS app

The loop

  1. Run the one-liner. It clones a pinned commit of these rails and creates a workspace that already contains a working iOS app.
  2. Start your coding agent in the workspace and hand it one sentence — the app you want.
  3. Run the app in the simulator, then on your phone.
{{ONESHOT_COMMAND}}

The bootstrap is a small, inspectable shell script pinned to an exact commit. It accepts no secrets, sends no telemetry, creates no accounts, and deploys nothing. Read it first if you prefer: curl -fsSL https://tohseno.com/oneshot.sh | less.

What a continuity app is

A continuity app is an app where cryptography replaces accounts. On first launch it silently generates a BIP39 seed phrase; a keypair derived from that phrase is the user's identity. Identity is backed up automatically through iCloud Keychain, end-to-end encrypted; the phrase is the manual fallback. Data lives on the device as plain files. There are no signup forms, no OAuth screens, no email capture — not for you, not for your users.

What the workspace contains

  • A compiling iOS app — a lightweight writing app demonstrating the full spine: open, write, it's yours. It builds from a fresh clone with only a signing-team selection and runs in the simulator with zero API keys.
  • Flag-gated modules — paywall (RevenueCat key slot), share card, notifications — present in code, flipped in one config file.
  • A packaged landing page — site/index.html ships with the app.
  • The agent protocol — skills/continuity-app/SKILL.md: your agent copies the base app and mutates it toward your prompt, asks at most three questions, records every assumed default, and ends with a completion report whose RUN IT NOW section is literal copy-paste commands.
  • The manifest — a machine-readable record of what was built. If a feature can't be expressed as a valid manifest field, it's unsupported and the agent says so instead of improvising.

To your phone

bun run setup in the workspace asks for your app name, bundle ID, and Apple Team ID (one question at a time; App Store Connect key and RevenueCat key are optional), then writes the config the Xcode project reads. A prepared fastlane lane uploads to TestFlight — your agent prints the command and never runs it without your approval.

Everything else

Full documentation, doctrine, the manifest schema, and the source live in the repository. Apache-2.0. Every app is ejectable from birth: it builds and runs without any TOHSENO credential.
