# Public intent distillations

This directory demonstrates how TOHSENO can preserve product direction without
committing private owner prompts. Entries are sanitized, owner-reviewable
distillations of a change request. They are not the canonical private intent
artifact and do not authorize implementation or deployment.

The future private lineage is described in
[Application evolution history](../proposals/APPLICATION_EVOLUTION_HISTORY.md).
Until that contract is implemented:

- do not paste raw customer or owner prompts here by default;
- do not include contact details, credentials, capabilities, private content,
  production data, or secret-bearing references;
- record unresolved thoughts as unresolved rather than completing them for the
  owner;
- trace accepted work through a manifest diff, code commit, tests, and release
  evidence;
- publish tutorial derivatives only through a separate owner approval.

The first record captures the software-seed, phone/browser bridge, and optional
external-rail direction that established this convention. The second record
captures the one-package deployment-cell requirement and the clarified
phone-rooted authorization invariant, and names its parent record explicitly.
The third record captures the `curl | bash` oneshot terminal entry and its
pinned two-stage trust design.
